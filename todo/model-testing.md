# Тестирование качества моделей

Заметки по исследованию (планирование, не реализация). Отдельный харнесс для массового
тестирования качества ответов моделей — с возможностью экспериментов на любых моделях,
а не только на тех, что зашиты в приложении.

## Где запускать — отдельный CLI-скрипт, не в приложении

- Приложение поставляет кураторскую, безопасную конфигурацию. Харнесс — противоположность:
  открытый перебор любых моделей. Разный жизненный цикл.
- Эвалу нужны batch-прогоны, CSV/markdown-отчёты, параллелизм, ретраи, трекинг стоимости —
  это CLI-территория, не GUI.
- Не тащить в шипнутый бинарь код с доступом к произвольным моделям и кучей ключей.

Конкретно: `scripts/eval/` или `tools/eval/` в этом же репо, переиспользует TS-типы приложения,
запускается как Node-CLI. Читает `cases.*` (тест-кейсы) и `models.*` (любые модели с любыми
параметрами), гоняет декартово произведение, пишет отчёт.

## Шаблонные объекты-параметры

База-шаблон с `{{плейсхолдерами}}` + матрица переменных + матрица моделей:

```
{
  name: "filler-removal-ru",
  template: {
    messages: [
      { role: "system", content: "{{SYSTEM_PROMPT}}" },
      { role: "user", content: "{{INPUT}}" }
    ],
    params: { temperature: 0, max_tokens: "{{MAX}}" }
  },
  vars: { INPUT: "ну эээ давай запушим", MAX: 500, SYSTEM_PROMPT: "..." },
  assertions: [
    { type: "not-contains", value: "эээ" },
    { type: "length-ratio-max", value: 1.2 },
    { type: "not-matches", value: "<think>" }
  ]
}
```

## Тест-кейсы

Самое важное — антигаллюцинационные. Ассерты property-based (regex, отношение длин,
«не содержит X») + LLM-судья для семантики (кейсы 3, 6, 9), судью фиксируем на temperature=0.

| #   | Кейс                                                | Что проверяет                  | Тип ассерта              |
| --- | --------------------------------------------------- | ------------------------------ | ------------------------ |
| 1   | Удаление филлеров («эээ», «ну», «как бы»)           | чистка без потери смысла       | not-contains             |
| 2   | Восстановление пунктуации/регистра                  | STT отдаёт lowercase без точек | regex                    |
| 3   | Без галлюцинаций — модель не дописывает предложения | критично                       | length-ratio + LLM-judge |
| 4   | Без thinking-утечки (`<think>…`)                    | чистота вывода                 | not-matches              |
| 5   | Без мета-комментариев («Вот исправленный текст:»)   | модель не болтает              | not-matches              |
| 6   | Сохранение языка (ru не переводить в en)            |                                | LLM-judge                |
| 7   | Коррекция по словарю («Таури» → «Tauri»)            | фича словаря                   | contains                 |
| 8   | Дисциплина длины (вывод ≈ длине входа)              | ловит runaway-генерацию        | length-ratio-max         |
| 9   | Code-switching («запушь в main branch»)             | смешанный язык                 | LLM-judge                |
| 10  | Числа/единицы («двадцать процентов» → «20 %»)       | по вкусу                       | contains                 |
| 11  | Идемпотентность (двойной прогон стабилен)           |                                | сравнение прогонов       |
| 12  | Пустой / почти пустой вход                          | graceful                       | not-error                |
| 13  | Очень длинный вход                                  | лимиты токенов                 | not-error                |

## promptfoo — build vs buy

`promptfoo` — open-source CLI (npm-пакет `promptfoo`), который гоняет матрицу
«промпты × модели × тест-кейсы». Это, по сути, та терминальная программа, которую иначе
пришлось бы писать самим. Почти все хотелки — его встроенные концепции.

| Идея                                 | Концепция promptfoo                                        |
| ------------------------------------ | ---------------------------------------------------------- |
| Шаблонный объект-параметр с вставкой | `prompts` с Nunjucks-плейсхолдерами                        |
| «Любые модели, не зашитые»           | `providers` — OpenAI, OpenRouter, Groq, Ollama, любой HTTP |
| Тест-кейсы с проверками              | `tests` + `assert` (детерминированные и model-graded)      |
| «Где запускать»                      | `promptfoo eval` в терминале + веб-UI для сравнения        |
| «Не хочу платить»                    | провайдеры `ollama:` (локально) и OpenRouter `:free`       |

Пример `promptfooconfig.yaml`:

```yaml
description: 'Transcription post-processing quality'

prompts:
  - file://prompts/cleanup.json # системка + {{input}} в chat-формате

providers:
  - id: openai:gpt-4o-mini
    config: { temperature: 0, max_tokens: 500 }
  - id: openrouter:anthropic/claude-haiku-4.5
    config: { temperature: 0, max_tokens: 500 }
  - id: groq:llama-3.3-70b-versatile
    config: { temperature: 0 }
  - id: ollama:chat:llama3.1 # локально, бесплатно

tests:
  - vars:
      input: 'ну эээ давай это запушим в мейн как бы'
    assert:
      - { type: not-icontains, value: 'эээ' }
      - { type: not-contains, value: '<think>' }
      - { type: not-icontains, value: 'вот исправленный' }
      - type: javascript
        value: output.length <= vars.input.length * 1.3
      - type: llm-rubric
        value: 'Сохраняет смысл, ничего не добавляет и не переводит'
```

`promptfoo eval` прогоняет все модели × все кейсы, кеширует, считает стоимость и латентность.
`promptfoo view` открывает веб-таблицу: строки — кейсы, колонки — модели, в ячейках вывод +
pass/fail. Это side-by-side сравнение — главная ценность.

**Что promptfoo НЕ закрывает:** сам STT (подача аудиофайла). promptfoo заточен под
text-in/text-out; тест транскрипции с аудио из коробки не делает, нужен кастомный
HTTP/python-провайдер. Силён для постобработки (LLM), для оценки самой расшифровки — свой скрипт.

**Вердикт:** для матрицы экспериментов по постобработке — брать promptfoo, не писать своё
(покрывает ~90%: шаблоны, мультипровайдер, ассерты, судья, сравнение, стоимость). Самописный
харнесс осмыслен только для аудио-части STT.

## Дешёвые эксперименты

Чтобы не платить за API при массовых прогонах:

- OpenRouter `:free`-варианты моделей (`deepseek/deepseek-chat-v3:free` и т.п.) — нулевая стоимость.
- Локальные модели через Ollama / LM Studio — OpenAI-совместимый эндпоинт, платить не надо.

Ходить в модель через подписку кодинг-агента (Claude Code / Codex) для этого не годится: теряется
контроль над параметрами (`temperature`, `response_format`, отключение thinking), и это против ToS.
