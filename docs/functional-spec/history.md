# История

## Назначение

История хранит результаты диктовок и позволяет пользователю возвращаться к аудио, транскрипции, постобработке и ошибкам прошлых запусков.

## Список истории

Главный экран приложения открывает историю. Записи сгруппированы по дням внутри выбранного месяца.

Пользователь может:

- выбрать месяц через календарь;
- перейти к предыдущему или следующему месяцу;
- вернуться к текущему месяцу;
- раскрыть группу дня;
- выбрать запись для просмотра деталей.

Если в выбранном месяце записей нет, отображается пустое состояние.

Записи сортируются от новых к старым. Внутри записи в списке показывается время и текст для отображения.

Правила выбора текста для отображения:

- если постобработка успешна, показывается текст постобработки;
- иначе если распознавание успешно, показывается текст распознавания;
- иначе показывается сообщение об ошибке распознавания, если оно есть.

## Быстрые действия в списке

Для каждой записи доступны действия:

- скопировать текст;
- повторить обработку записи;
- удалить запись.

Копирование доступно только если у записи есть успешная постобработка или успешная транскрипция.

Повтор из списка запускает полный повтор записи: сначала заново выполняется распознавание по сохраненному аудио с текущими настройками Speech-to-Text, затем, если постобработка сейчас включена и распознавание успешно, заново выполняется постобработка с текущими настройками постобработки.

Пока запись обрабатывается, повторное действие для нее недоступно и отображается состояние обработки.

Удаление записи удаляет ее из истории и удаляет связанный аудиофайл, если он существует.

## Детали записи

При выборе записи открывается панель деталей. В ней показаны:

- длительность аудио;
- действия с аудио;
- результат распознавания;
- результат постобработки, если он был выполнен или может быть выполнен;
- провайдер, модель, длительность и стоимость этапов, если данные доступны;
- текст результата или сообщение об ошибке.

Доступные действия:

- закрыть панель;
- скопировать путь к аудиофайлу;
- открыть аудиофайл в проводнике;
- скопировать текст распознавания;
- повторить только распознавание;
- скопировать текст постобработки;
- повторить только постобработку.

Открытие аудио в проводнике выделяет сохраненный файл. Если файл не найден, пользователь получает ошибку.

## Связь с отладочными логами

Если отладочное логирование включено, события обработки записи содержат идентификатор записи истории и время начала аудиозаписи.

По этим данным пользователь может:

- найти конкретную запись в истории по идентификатору записи;
- сверить время начала записи с временем создания записи истории;
- сопоставить запросы и ответы Speech-to-Text и постобработки с конкретной аудиозаписью.

## Повтор только распознавания

Повтор распознавания использует сохраненный аудиофайл и текущие настройки Speech-to-Text.

При запуске:

- результат распознавания переводится в состояние обработки;
- статус всей записи становится `processing`;
- после завершения сохраняется новый результат или ошибка;
- итоговый текст пересчитывается.

Постобработка при этом не запускается автоматически. Если в записи уже был успешный результат постобработки, итоговый текст может продолжать опираться на него до отдельного повтора постобработки или полного повтора записи.

## Повтор только постобработки

Повтор постобработки доступен, если:

- постобработка сейчас включена в настройках;
- у записи есть успешная транскрипция.

При запуске используется текущий текст транскрипции и текущие настройки постобработки. После завершения сохраняется новый результат или ошибка, а итоговый текст пересчитывается.

Если транскрипции нет или она завершилась ошибкой, повтор постобработки невозможен. Если постобработка выключена, повтор постобработки невозможен.

## Статусы

Статус этапа может быть:

- `success` - этап успешно завершен;
- `error` - этап завершился ошибкой;
- `processing` - этап выполняется;
- `skipped` - этап не выполнялся.

Статус записи может быть:

- `success` - распознавание успешно, постобработка не завершилась ошибкой;
- `error` - распознавание неуспешно или постобработка завершилась ошибкой;
- `processing` - распознавание или постобработка сейчас выполняется.

Итоговый текст записи выбирается так:

- успешная постобработка имеет приоритет;
- если успешной постобработки нет, используется успешная транскрипция;
- если успешных результатов нет, итоговый текст пустой.

## Обновление истории

Когда приложение создает, изменяет или удаляет запись, экран истории получает событие обновления и заново загружает данные выбранного месяца.
