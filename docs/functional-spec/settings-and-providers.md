# Настройки, провайдеры и модели

## Общие настройки

Пользователь может изменить:

- тему интерфейса: светлая, темная или системная;
- язык интерфейса: системный, русский или английский;
- заглушать звук при записи: включено или выключено;
- запуск при входе в Windows: включен или выключен;
- отладочное логирование: включено или выключено;
- горячую клавишу диктовки;
- горячую клавишу отмены диктовки;
- режим запуска записи: по нажатию или по зажатию.

Значения по умолчанию:

- тема: светлая;
- язык интерфейса: системный;
- эффективный язык интерфейса: русский для системной локали `ru*`, иначе английский;
- заглушать звук при записи: включено;
- запуск при входе в Windows: включен;
- отладочное логирование: выключено;
- горячая клавиша диктовки: `Ctrl+Space`;
- горячая клавиша отмены: `Ctrl+Z`;
- режим запуска: по нажатию.

Горячие клавиши диктовки и отмены задаются через захват комбинации: пользователь нажимает кнопку рядом с полем, после чего приложение переходит в режим записи и считывает первое нажатие клавиши (с модификаторами). На время режима записи горячая клавиша диктовки не активна. Поле с текущей комбинацией доступно только для чтения — ручной ввод невозможен. При изменении горячей клавиши диктовки или отмены приложение нормализует ее. Если комбинация не поддерживается, настройка не сохраняется и пользователь получает ошибку.

Если запуск при входе в Windows включен, приложение регистрирует автозапуск текущего пользователя и стартует после входа в систему в фоновом режиме без открытия главного окна. Если настройка выключена, автозапуск удаляется. Ручной запуск приложения открывает главное окно.

## Фоновый режим

Приложение может работать без открытого главного окна. В фоновом режиме продолжают работать горячая клавиша диктовки, запись, обработка, вставка текста и обновление истории.

Когда пользователь закрывает главное окно через системную кнопку закрытия, приложение скрывает окно и остается запущенным. Полное завершение выполняется через пункт `Выход` в меню tray-иконки.

В системной области Windows приложение показывает иконку Transcriber. Левый клик по иконке открывает главное окно. Правый клик открывает системное меню с пунктами:

- `Открыть приложение`;
- `Скопировать последнюю расшифровку`;
- `Выход`.

Пункт копирования последней расшифровки доступен только если последняя запись истории имеет итоговый текст. Если истории нет, последняя запись еще обрабатывается или завершилась ошибкой распознавания без итогового текста, пункт недоступен.

## Отладочное логирование

Пользователь может включить локальное отладочное логирование в разделе общих настроек.

Когда логирование включено:

- приложение сохраняет локальные логи запросов и ответов Speech-to-Text и постобработки;
- логи не отправляются на сервер и нужны только для отладки;
- под переключателем отображается действие открытия папки логов;
- при открытии папки логов приложение выделяет текущий файл логов, если он уже создан.

Логи содержат полные параметры запросов, промпты, ответы модели, usage, стоимость, статусы и ошибки. API-ключи, `Authorization`, аудиобайты и значения дополнительных заголовков в логи не записываются.

Каждый запуск приложения использует отдельный файл логов. Если пользователь выключает логирование и затем включает его снова в рамках того же запуска, приложение создает новый файл логов.

## Провайдеры

Провайдер - сохраненная конфигурация подключения к API моделей. Поддержаны типы:

- OpenAI;
- Groq;
- OpenRouter;
- Custom.

Для OpenAI, Groq и OpenRouter приложение знает стандартный базовый URL. Для Custom URL обязателен.

При добавлении провайдера пользователь выбирает тип, может указать отображаемое имя, вводит API-ключ или токен и при необходимости задает дополнительные параметры.

Правила сохранения:

- API-ключ обязателен при создании провайдера;
- при редактировании сохраненный ключ можно оставить без изменений;
- если имя не указано, используется имя типа провайдера;
- для стандартных провайдеров дополнительные URL и заголовки применяются только при включенных расширенных настройках;
- для Custom расширенные настройки считаются включенными всегда;
- сохраненный ключ в интерфейсе показывается только в маскированном виде.

Удаление провайдера удаляет его конфигурацию. Текущее поведение не очищает автоматически выбранные ссылки на удаленного провайдера в настройках обработки; если такая ссылка останется, последующий запуск обработки завершится ошибкой поиска провайдера.

## Проверка провайдера и список моделей

В карточке провайдера пользователь может:

- проверить конфигурацию;
- запросить список моделей;
- сравнить полученный список с моделями из встроенного каталога.

Проверка конфигурации выполняет запрос списка моделей. Если запрос успешен, пользователь видит сообщение с количеством найденных моделей. Если запрос завершился ошибкой, пользователь видит текст ошибки.

Список моделей показывает модели из встроенного каталога для выбранного типа провайдера. После получения ответа провайдера приложение помечает, какие из каталожных моделей реально доступны по API-идентификатору.

## Расширенные параметры провайдера

Расширенные параметры включают:

- пользовательский базовый URL;
- дополнительные HTTP-заголовки.

Заголовки вводятся построчно в формате `Name: value`. Пустые строки игнорируются. Строка без двоеточия или с некорректным именем/значением заголовка считается ошибкой.

## Каталог моделей

Приложение использует встроенный каталог моделей. Модели разделены на две задачи:

- Speech-to-Text;
- Post-processing.

Для Speech-to-Text в текущем каталоге доступны:

- GPT-4o Transcribe через OpenAI;
- GPT-4o mini Transcribe через OpenAI;
- Whisper Large v3 через Groq;
- Whisper Large v3 Turbo через Groq.

Для постобработки в текущем каталоге доступны:

- Claude Haiku 4.5 через OpenRouter;
- GPT-4o mini через OpenAI и OpenRouter;
- GPT-4.1 mini через OpenAI и OpenRouter;
- GPT-5 mini через OpenAI и OpenRouter;
- GPT-5.4 mini через OpenAI и OpenRouter;
- GPT OSS 120B через Groq и OpenRouter;
- Qwen 3.6 27B через Groq и OpenRouter;
- Gemini 2.5 Flash через OpenRouter;
- Gemini 2.5 Flash Lite через OpenRouter;
- Gemini 3.1 Flash Lite через OpenRouter;
- Llama 4 Scout через Groq и OpenRouter.

Часть моделей помечена как нерекомендуемая. В настройках такие модели показываются отдельно в группе нерекомендуемых.

## Настройки Speech-to-Text

Пользователь выбирает провайдера, модель и язык распознавания.

В списке провайдеров показываются только сохраненные провайдеры, для которых в каталоге есть хотя бы одна модель Speech-to-Text. Если подходящих провайдеров нет, пользователь видит пустое состояние с предложением добавить провайдера.

В списке моделей показываются только модели Speech-to-Text, совместимые с выбранным типом провайдера.

Если у пользователя есть совместимый провайдер и модель, но он еще не делал явный выбор, приложение автоматически сохраняет первый доступный провайдер и первую доступную модель как текущие значения.

Пользователь может включить пользовательский prompt. Если переключатель выключен, используется prompt по умолчанию. Если переключатель включён и поле пустое, prompt не передаётся в запрос.

## Настройки постобработки

Пользователь может включить или выключить постобработку.

При включенной постобработке пользователь выбирает провайдера и модель из совместимых с задачей Post-processing. Если совместимых провайдеров нет, отображается пустое состояние.

Для постобработки можно включить пользовательские prompts:

- системный prompt;
- шаблон пользовательского prompt.

Если пользовательские prompts выключены, используются значения по умолчанию. Если пользовательские prompts включены и системный prompt пустой, системный prompt не передаётся в запрос. Если пользовательские prompts включены и шаблон пользовательского prompt пустой, используется шаблон по умолчанию. В шаблон пользовательского prompt подставляется распознанный текст.

## Тест Speech-to-Text

В настройках Speech-to-Text есть тестовая панель. Она работает только если выбраны провайдер и модель.

Пользователь может:

- записать голос через браузерный доступ к микрофону;
- загрузить аудиофайл;
- повторить последний тест с тем же аудио.

Поддерживаются аудиофайлы, распознаваемые как `audio/*`, а также расширения `.mp3`, `.wav`, `.ogg`, `.flac`, `.webm`, `.m4a`.

После запуска теста пользователь видит результат или ошибку, а также длительность выполнения. Тест не создает запись в истории и не вставляет текст в активное приложение.

## Тест постобработки

В настройках постобработки есть тестовая панель. Она работает только если выбраны провайдер и модель, а пользователь ввел непустой текст.

Пользователь вводит текст, запускает обработку и получает результат или ошибку с длительностью выполнения. Тест не создает запись в истории и не меняет исходный текст вне панели теста.
