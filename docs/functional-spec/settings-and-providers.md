# Настройки, провайдеры и модели

## Общие настройки

На вкладке «Основное» пользователь может изменить:

- режим запуска записи: по нажатию или по зажатию;
- звук при записи: заглушать вывод, ставить воспроизведение на паузу или не трогать звук;
- возвращать звук во время паузы записи: включено или выключено. Настройка не показывается, когда звук при записи не трогается; в режиме запуска по зажатию она остается видимой, но заблокирована, а под описанием показывается желтое сообщение о недоступности;
- запуск при входе в Windows: включен или выключен;
- отладочное логирование: включено или выключено;

На вкладке «Дизайн» пользователь может изменить:

- тему интерфейса: светлая, темная или системная;
- язык интерфейса: системный, русский или английский;
- вид оверлея записи: компактная панель снизу экрана или большая панель по центру;
- экраны для оверлея записи: только экран с курсором или каждый экран.

На вкладке «Хоткеи» пользователь может изменить:

- горячую клавишу диктовки;
- горячую клавишу отмены диктовки;
- горячую клавишу копирования последней расшифровки;
- горячую клавишу вставки последней расшифровки;
- горячую клавишу повторной обработки последней расшифровки;
- горячую клавишу паузы записи.

Значения по умолчанию:

- тема: светлая;
- язык интерфейса: системный;
- эффективный язык интерфейса: русский для системной локали `ru*`, иначе английский;
- звук при записи: заглушать вывод;
- возвращать звук во время паузы записи: включено;
- запуск при входе в Windows: включен;
- отладочное логирование: выключено;
- горячая клавиша диктовки: `Ctrl+Space`;
- режим запуска: по нажатию;
- горячая клавиша паузы записи: не задана;
- горячая клавиша отмены: `Ctrl+Z`;
- горячая клавиша копирования последней расшифровки: не задана;
- горячая клавиша вставки последней расшифровки: не задана;
- горячая клавиша повторной обработки последней расшифровки: не задана;
- вид оверлея записи: большая панель по центру;
- экраны для оверлея записи: каждый экран.

Горячие клавиши задаются через захват комбинации: пользователь нажимает кнопку рядом с полем, после чего приложение переходит в режим записи и считывает первое нажатие клавиши (с модификаторами). На время режима записи горячие клавиши диктовки, копирования последней расшифровки, вставки последней расшифровки и повторной обработки не активны. Поле с текущей комбинацией доступно только для чтения — ручной ввод невозможен. При изменении горячей клавиши приложение нормализует ее. Если комбинация не поддерживается, настройка не сохраняется и пользователь получает ошибку.

На вкладке «Основное» режим запуска записи расположен над настройкой звука при записи. На вкладке «Хоткеи» настройки отображаются в таком порядке: горячая клавиша запуска записи, горячая клавиша паузы записи, горячая клавиша отмены диктовки, горячая клавиша копирования последней расшифровки, горячая клавиша вставки последней расшифровки и горячая клавиша повторной обработки последней расшифровки.

У каждой горячей клавиши рядом с полем есть отдельная кнопка сброса, расположенная между полем и кнопкой переназначения. Кнопка сброса показывается только если текущее значение отличается от значения после сброса. Для горячей клавиши диктовки сброс возвращает значение по умолчанию `Ctrl+Space`. Для горячей клавиши отмены сброс возвращает `Ctrl+Z`. Для горячих клавиш паузы записи, копирования последней расшифровки, вставки последней расшифровки и повторной обработки сброс очищает значение. Во время записи новой комбинации кнопка сброса не показывается.

Горячие клавиши паузы записи, отмены, копирования последней расшифровки, вставки последней расшифровки и повторной обработки можно оставить пустыми. В этом состоянии соответствующее действие отключено, а в поле настроек отображается текст о том, что хоткей не используется.

Горячая клавиша паузы записи доступна только при режиме запуска `По нажатию`. Если выбран режим `По зажатию`, поле этой горячей клавиши заблокировано: комбинацию нельзя записать или сбросить, а сохраненное ранее значение остается видимым и не срабатывает. Под описанием настройки показывается желтое сообщение о недоступности в режиме `По зажатию`. Поведение паузы описано в [Диктовка и обработка](dictation.md).

Приложение различает левые и правые модификаторы. Если при записи комбинации была нажата только левая клавиша модификатора (например, левый Ctrl), приложение сохраняет ее как «левый Ctrl» и срабатывает только при нажатии именно левого Ctrl (правый должен быть отпущен). Если были нажаты обе стороны одновременно — сохраняется «любой» модификатор и срабатывает при нажатии любой стороны. Правило применяется к Ctrl, Alt, Shift и Win. Существующие настройки без явного указания стороны трактуются как «любой» и работают без изменений.

Если запуск при входе в Windows включен, приложение регистрирует автозапуск текущего пользователя и стартует после входа в систему в фоновом режиме без открытия главного окна. Если настройка выключена, автозапуск удаляется. Ручной запуск приложения открывает главное окно.

## Фоновый режим

Приложение может работать без открытого главного окна. В фоновом режиме продолжают работать горячие клавиши, запись, обработка, вставка текста и обновление истории.

Когда пользователь закрывает главное окно через системную кнопку закрытия, приложение скрывает окно и остается запущенным. Полное завершение выполняется через пункт `Выход` в меню tray-иконки.

Пока главное окно в фокусе, комбинация `Ctrl+W` закрывает его так же, как системная кнопка закрытия: окно скрывается в трей, приложение продолжает работать. Комбинация не перехватывается глобально и не работает, когда фокус находится в другом приложении. Если пользователь назначил `Ctrl+W` одной из настраиваемых горячих клавиш, приоритет у нее: окно не закрывается. `Ctrl+W` также не закрывает окно, пока в настройках идет запись новой комбинации горячей клавиши.

В системной области Windows приложение показывает иконку Transcriber. Левый клик по иконке переключает главное окно:

- если окно скрыто в трее или свернуто, приложение показывает окно, разворачивает его и передает ему фокус;
- если окно открыто, но находится на другом виртуальном рабочем столе Windows, приложение переводит фокус в это окно; система переключается на его рабочий стол, а само окно не скрывается;
- если окно открыто на текущем виртуальном рабочем столе, приложение скрывает его в трей. Это происходит и тогда, когда окно перекрыто другими окнами и не имеет фокуса.

Монитор, на котором открыто окно, на это поведение не влияет: окно на соседнем мониторе пользователь видит, поэтому оно скрывается как обычно.

Правый клик открывает системное меню с пунктами:

- `Открыть приложение`;
- `Скопировать последнюю расшифровку`;
- `Выход`.

Пункт `Открыть приложение` всегда открывает главное окно и никогда его не скрывает.

Пункт копирования последней расшифровки доступен только если последняя запись истории имеет итоговый текст. Если истории нет, последняя запись еще обрабатывается или завершилась ошибкой распознавания без итогового текста, пункт недоступен.

Горячая клавиша копирования последней расшифровки копирует итоговый текст из последней записи истории в обычный буфер обмена. После этого текст доступен для стандартной вставки и виден в истории буфера обмена Windows. Если последней записи нет или в ней нет итогового текста, горячая клавиша ничего не делает.

Горячая клавиша вставки последней расшифровки вставляет итоговый текст из последней записи истории в активное текстовое поле без сохранения этого текста как обычного содержимого буфера обмена. Если последней записи нет, в ней нет итогового текста или в приложении уже идет запись либо обработка, горячая клавиша ничего не делает.

Горячая клавиша повторной обработки последней расшифровки повторно обрабатывает последнюю запись истории, сразу показывает оверлей стадии обработки и после успешного завершения вставляет итоговый текст в активное текстовое поле. Эта горячая клавиша ничего не делает, если последней записи нет или в приложении уже идет запись либо обработка. Во время такой повторной обработки работает горячая клавиша отмены диктовки.

## Отладочное логирование

Пользователь может включить локальное отладочное логирование в разделе общих настроек.

Когда логирование включено:

- приложение сохраняет локальные логи запросов и ответов Speech-to-Text и постобработки;
- логи не отправляются на сервер и нужны только для отладки;
- под переключателем отображается действие открытия папки логов;
- при открытии папки логов приложение выделяет текущий файл логов, если он уже создан.

Логи содержат полные параметры запросов, промпты, ответы модели, usage, стоимость, статусы и ошибки. API-ключи, `Authorization`, аудиобайты и значения дополнительных заголовков в логи не записываются.

Каждый запуск приложения использует отдельный файл логов. Если пользователь выключает логирование и затем включает его снова в рамках того же запуска, приложение создает новый файл логов.

## Провайдеры

Провайдер - сохраненная конфигурация подключения к API моделей. Поддержаны типы:

- OpenAI;
- Groq;
- OpenRouter;
- Custom.

Для OpenAI, Groq и OpenRouter приложение знает стандартный базовый URL. Для Custom URL обязателен.

При добавлении провайдера пользователь выбирает тип, может указать отображаемое имя, вводит API-ключ или токен и при необходимости задает дополнительные параметры.

Правила сохранения:

- API-ключ обязателен при создании провайдера;
- при редактировании сохраненный ключ можно оставить без изменений;
- если имя не указано, используется имя типа провайдера;
- для стандартных провайдеров дополнительные URL и заголовки применяются только при включенных расширенных настройках;
- для Custom расширенные настройки считаются включенными всегда;
- сохраненный ключ в интерфейсе показывается только в маскированном виде.

Удаление провайдера удаляет его конфигурацию. Если этот провайдер был выбран в настройках Speech-to-Text или постобработки, приложение очищает связанный выбор провайдера и модели, чтобы пользователь заново выбрал совместимую конфигурацию.

## Проверка провайдера и список моделей

В карточке провайдера пользователь может:

- проверить конфигурацию;
- запросить список моделей;
- сравнить полученный список с моделями из встроенного каталога.

Проверка конфигурации выполняет запрос списка моделей. Если запрос успешен, пользователь видит сообщение с количеством найденных моделей. Если запрос завершился ошибкой, пользователь видит текст ошибки.

Список моделей показывает модели из встроенного каталога для выбранного типа провайдера. После получения ответа провайдера приложение помечает, какие из каталожных моделей реально доступны по API-идентификатору.

## Расширенные параметры провайдера

Расширенные параметры включают:

- пользовательский базовый URL;
- дополнительные HTTP-заголовки.

Заголовки вводятся построчно в формате `Name: value`. Пустые строки игнорируются. Строка без двоеточия или с некорректным именем/значением заголовка считается ошибкой.

## Каталог моделей

Приложение использует встроенный каталог моделей. Модели разделены на две задачи:

- Speech-to-Text;
- Post-processing.

Для Speech-to-Text в текущем каталоге доступны:

- GPT-4o Transcribe через OpenAI;
- GPT-4o mini Transcribe через OpenAI;
- Whisper Large v3 через Groq;
- Whisper Large v3 Turbo через Groq.

Для постобработки в текущем каталоге доступны:

- Claude Haiku 4.5 через OpenRouter;
- GPT-4o mini через OpenAI и OpenRouter;
- GPT-4.1 mini через OpenAI и OpenRouter;
- GPT-5 mini через OpenAI и OpenRouter;
- GPT-5.4 mini через OpenAI и OpenRouter;
- GPT OSS 120B через Groq и OpenRouter;
- Qwen 3.6 27B через Groq и OpenRouter;
- Qwen 3.6 35B A3B через OpenRouter;
- Gemini 2.5 Flash через OpenRouter;
- Gemini 2.5 Flash Lite через OpenRouter;
- Gemini 3.1 Flash Lite через OpenRouter;
- Llama 4 Scout через Groq и OpenRouter.

Часть моделей помечена как нерекомендуемая. В настройках такие модели показываются отдельно в группе нерекомендуемых.

## Настройки Speech-to-Text

Пользователь выбирает провайдера, модель и язык распознавания.

В списке провайдеров показываются только сохраненные провайдеры, для которых в каталоге есть хотя бы одна модель Speech-to-Text. Если подходящих провайдеров нет, пользователь видит пустое состояние с предложением добавить провайдера.

В списке моделей показываются только модели Speech-to-Text, совместимые с выбранным типом провайдера.

Если у пользователя есть совместимый провайдер и модель, но он еще не делал явный выбор, приложение автоматически сохраняет первый доступный провайдер и первую доступную модель как текущие значения.

Если выбранная модель перестала быть совместимой с текущим провайдером или выбранный провайдер больше недоступен, приложение очищает невалидный выбор. Пользователь должен выбрать совместимую конфигурацию заново, прежде чем запускать тест или диктовку.

Если обязательный выбор провайдера или модели отсутствует, соответствующее поле в настройках Speech-to-Text сразу подсвечивается как ошибка красной рамкой.

Пользователь может включить пользовательский prompt. Если переключатель выключен, используется prompt по умолчанию. Если для пользовательского prompt не задано сохранённое значение, в поле настроек показывается prompt по умолчанию и в запросах используется prompt по умолчанию. Если пользователь ввёл любой текст, включая пустую строку, используется именно это значение и оно сохраняется при выключении и повторном включении переключателя. Пока пользовательский prompt включён, рядом с полем доступна кнопка сброса. Сброс удаляет сохранённое пользовательское значение, после чего в поле снова показывается prompt по умолчанию и в запросах снова используется prompt по умолчанию.

## Настройки постобработки

Пользователь может включить или выключить постобработку.

Когда постобработка выключена, элементы выбора провайдера и модели, настройки prompts и тестовая панель постобработки не показываются.

При включенной постобработке пользователь выбирает провайдера и модель из совместимых с задачей Post-processing. Если совместимых провайдеров нет, отображается пустое состояние.

Если выбранная модель постобработки перестала быть совместимой с текущим провайдером или выбранный провайдер больше недоступен, приложение очищает невалидный выбор. Пока новая совместимая конфигурация не выбрана, тест постобработки и сама постобработка недоступны.

Если обязательный выбор провайдера или модели отсутствует, соответствующее поле в настройках постобработки сразу подсвечивается как ошибка красной рамкой.

## Выбор апстрим-провайдера OpenRouter

Когда для постобработки выбран провайдер типа OpenRouter, под выбором модели появляется дополнительное поле выбора апстрим-провайдера — конкретной инфраструктуры, которая фактически выполнит запрос за OpenRouter.

Значение по умолчанию — «Авто»: OpenRouter сам выбирает апстрим-провайдера для запроса. Список остальных вариантов запрашивается у OpenRouter для конкретной выбранной модели и может отличаться от модели к модели.

Один и тот же провайдер может отдавать модель через несколько вариантов, отличающихся регионом или квантизацией. Такие варианты показываются как отдельные пункты списка, а уточнение добавляется к названию провайдера, например `Google (eu)` и `Google (global)`. Варианты, отличающиеся только сервисным тарифом, отдельными пунктами не показываются.

Список отсортирован по названию провайдера; пункт «Авто» всегда показывается первым независимо от сортировки. Поле поддерживает поиск: ввод текста фильтрует список по названию.

Если выбран конкретный апстрим-провайдер, он используется с приоритетом, но при его недоступности OpenRouter автоматически переключается на другого доступного провайдера для той же модели — запрос не завершается ошибкой только из-за недоступности предпочтённого провайдера.

Выбор апстрим-провайдера сбрасывается на «Авто», если пользователь меняет провайдера или модель постобработки: набор доступных апстрим-провайдеров привязан к конкретной модели.

Поле выбора апстрим-провайдера не отображается, если провайдер постобработки не является OpenRouter.

Если запрос списка апстрим-провайдеров у OpenRouter завершился ошибкой, поле остаётся доступным только со значением «Авто», без отдельного сообщения об ошибке.

Для постобработки можно включить пользовательские prompts:

- системный prompt;
- шаблон пользовательского prompt.

Если пользовательские prompts выключены, используются значения по умолчанию. Если для системного prompt или шаблона пользовательского prompt не задано сохранённое значение, в соответствующем поле настроек показывается значение по умолчанию и в запросах используется значение по умолчанию. Если пользователь ввёл любой текст в системный prompt или шаблон пользовательского prompt, включая пустую строку, используется именно это значение и оно сохраняется при выключении и повторном включении переключателя. Пока пользовательские prompts включены, рядом с полями системного prompt и шаблона пользовательского prompt доступны кнопки сброса. Сброс удаляет сохранённое пользовательское значение для соответствующего поля, после чего в поле снова показывается значение по умолчанию и в запросах снова используется значение по умолчанию. В шаблон пользовательского prompt подставляется распознанный текст.

## Тест Speech-to-Text

В настройках Speech-to-Text есть тестовая панель. Она работает только если выбраны провайдер и модель.

Пользователь может:

- записать голос через браузерный доступ к микрофону;
- загрузить аудиофайл;
- повторить последний тест с тем же аудио.

Поддерживаются аудиофайлы, распознаваемые как `audio/*`, а также расширения `.mp3`, `.wav`, `.ogg`, `.flac`, `.webm`, `.m4a`.

После запуска теста пользователь видит результат или ошибку, а также длительность выполнения. Тест не создает запись в истории и не вставляет текст в активное приложение.

## Тест постобработки

В настройках постобработки есть тестовая панель. Она работает только если выбраны провайдер и модель, а пользователь ввел непустой текст.

Пользователь вводит текст, запускает обработку и получает результат или ошибку с длительностью выполнения. Тест не создает запись в истории и не меняет исходный текст вне панели теста.

## О программе

В разделе «О программе» отображается текущая установленная версия приложения.

В нестабильных (canary) сборках рядом с версией отображается пометка «Canary». Такие сборки также
имеют отдельную иконку приложения, отличную от стабильной версии.

Пользователь может:

- проверить наличие обновлений вручную кнопкой «Проверить обновления»;
- включить или выключить внутриприкладное уведомление о новой версии переключателем «Уведомлять о новых версиях» (по умолчанию включен);
- включить получение нестабильных версий (альфа- и бета-релизов) переключателем «Предлагать нестабильные версии» (по умолчанию выключен);
- установить найденное обновление кнопкой «Установить X.Y.Z» с индикатором прогресса загрузки;
- сбросить данные приложения кнопкой «Сбросить данные» с подтверждением (см. [Состояния, ограничения и ошибки](errors-and-limits.md)).

При запуске приложения выполняется однократная автоматическая проверка обновлений после загрузки настроек, если включен переключатель «Уведомлять о новых версиях». Если обновление найдено, в правом нижнем углу появляется ненавязчивое уведомление с номером версии, кнопкой «Скачать» и встроенным индикатором времени на 10 секунд. Уведомление не содержит списка изменений новой версии — только номер версии и кнопку.

Это уведомление закрывается автоматически через 10 секунд, показывает progress bar этого таймера и ставит таймер на паузу, пока курсор находится над уведомлением. Нажатие на кнопку «Скачать» не начинает загрузку обновления, а открывает настройки на вкладке «О программе».

При открытии вкладки «О программе» приложение всегда заново проверяет наличие обновлений, даже если внутриприкладные уведомления выключены. Если обновление уже было найдено ранее, кнопка установки и номер версии показываются сразу, до завершения повторной проверки.

Если у найденного обновления есть описание изменений, во вкладке «О программе» под кнопкой установки всегда отображается развернутый блок «Что нового в X.Y.Z» со списком изменений этой версии. Разметка описания отображается форматированной: заголовки разделов, маркированные списки и выделения текста. Если описание изменений отсутствует или пустое, блок не показывается. Ссылки внутри описания не открываются по нажатию, их адрес виден во всплывающей подсказке.

Канал обновлений определяется настройкой «Предлагать нестабильные версии»: при выключенном переключателе проверяется стабильный канал, при включённом — нестабильный. Включённый нестабильный канал позволяет получить последнюю опубликованную версию вне зависимости от того, стабильная она или предварительная.

После скачивания обновления из вкладки «О программе» приложение перезапускается автоматически.
