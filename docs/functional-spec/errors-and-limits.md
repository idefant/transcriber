# Состояния, ограничения и ошибки

## Жизненный цикл приложения и фоновой режим

Приложение работает в одном экземпляре. Если пользователь пытается запустить его повторно, второй процесс не стартует: вместо этого окно уже работающего экземпляра выводится на передний план и получает фокус.

Закрытие главного окна не завершает приложение — оно скрывается в системный трей. Иконка в трее остаётся одна вне зависимости от количества попыток запуска. Выход из приложения выполняется через меню иконки трея.

При запуске с флагом `--hidden` (автозапуск при входе в систему) главное окно не показывается. Если при таком запуске приложение уже запущено, второй экземпляр завершается без каких-либо действий с окном.

## Глобальные состояния

Приложение загружает настройки, провайдеров и конфигурацию обработки при старте. Если загрузка не удалась, интерфейс сохраняет значения по умолчанию там, где это предусмотрено, а ошибка хранится в состоянии соответствующего раздела.

Основные состояния диктовки:

- неактивна;
- запись;
- распознавание;
- постобработка;
- отменена.

Основные состояния экранов:

- загрузка данных;
- пустое состояние;
- сохранение;
- обработка;
- ошибка.

## Сброс данных приложения

В настройках, на вкладке «О программе», доступно действие сброса данных приложения. Перед выполнением приложение запрашивает подтверждение.

При подтверждении все данные приложения — история, настройки, провайдеры и словарь — переносятся в резервную папку внутри каталога данных, после чего приложение перезапускается с чистого состояния. Данные не удаляются: они остаются в резервной папке, поэтому пользователь при необходимости может к ним вернуться.

## Данные, созданные более новой версией

Стабильные и предрелизные (canary) сборки используют один каталог данных. Если данные в этом каталоге были записаны более новой версией приложения, чем установленная сейчас (например, пользователь запускал предрелизную сборку, а затем вернулся на более старую стабильную), приложение не открывает эти данные, чтобы не повредить их.

Вместо обычного интерфейса показывается блокирующий экран с объяснением ситуации и двумя вариантами действий: обновить приложение до последней версии либо сбросить данные приложения (см. «Сброс данных приложения»). Пока данные несовместимы, диктовка и работа с историей недоступны.

Блокирующий экран сохраняет заголовок окна с названием приложения и кнопками сворачивания, разворачивания и закрытия, а само окно можно перетаскивать. Поэтому его всегда можно закрыть или переместить, даже если оно оказалось частично за границей экрана.

## Ошибки записи

Запись не начинается или завершается ошибкой, если:

- нет системного устройства ввода по умолчанию;
- не удалось прочитать конфигурацию устройства ввода;
- формат входного аудио не поддержан;
- не удалось запустить поток записи;
- не удалось прочитать записанные сэмплы;
- запись не содержит аудиоданных;
- не удалось создать или завершить WAV-файл.

При ошибке записи приложение отправляет пользователю сообщение ошибки и скрывает оверлей, если это возможно. При ошибке распознавания или постобработки приложение показывает уведомление об ошибке на оверлее (см. [Диктовка и обработка](dictation.md)).

## Ошибки конфигурации обработки

Обработка невозможна, если:

- не выбран провайдер;
- не выбрана модель;
- сохраненный провайдер не найден;
- у провайдера нет API-ключа;
- для Custom-провайдера не задан URL;
- модель отсутствует во встроенном каталоге;
- модель не поддерживает выбранный тип провайдера;
- выбранная модель относится не к той задаче.
- итоговый prompt Groq Whisper содержит больше 224 токенов.

Для основной диктовки готовность конфигурации проверяется до начала записи. Если обработать аудио нельзя (не выбран провайдер или модель, модель несовместима с провайдером, у провайдера нет API-ключа, а при включённой постобработке — те же условия для неё), запись не начинается, оверлей записи не показывается, а приложение показывает системное уведомление операционной системы с описанием проблемы. Клик по уведомлению открывает соответствующую вкладку настроек и закрывает уведомление. Такие системные уведомления доступны только на Windows (см. [Диктовка и обработка](dictation.md)).

Если итоговый prompt Groq Whisper превышает 224 токена, основная диктовка также не начинает запись и не отправляет частичный prompt. На Windows системное уведомление сообщает о превышении размера prompt без технических чисел, а клик по нему открывает главный экран сразу на странице «Словарь»; на других платформах запуск просто блокируется.

## Ошибки провайдера

Возможные ошибки при обращении к провайдеру:

- сетевой таймаут;
- недоступный API;
- ошибка авторизации;
- неуспешный HTTP-статус;
- ответ в неожиданном формате;
- пустой ответ модели;
- некорректный список моделей;
- некорректные дополнительные заголовки.

При неуспешном HTTP-статусе приложение сохраняет краткое сообщение со статусом ответа, а полное тело ответа доступно в деталях записи истории как раскрываемый блок (см. [История](history.md)).

## Ошибки истории

История может вернуть ошибку, если:

- запись не найдена;
- аудиофайл не найден;
- аудиофайл невозможно прочитать;
- аудиофайл невозможно удалить;
- не удалось открыть расположение аудио;
- не удалось сохранить обновленную историю.

Если повторная обработка записи завершается ошибкой, запись остается в истории и получает статус ошибки соответствующего этапа.

## Ошибки тестовых панелей

Тест Speech-to-Text может завершиться ошибкой, если:

- браузерный доступ к микрофону запрещен или недоступен;
- не удалось прочитать записанное аудио;
- не удалось прочитать загруженный файл;
- не выбраны провайдер и модель;
- провайдер вернул ошибку.

Тест постобработки невозможен без выбранного провайдера, модели и непустого входного текста.

Тестовые панели не создают записи истории и не вставляют текст в активное приложение.

## Платформенные ограничения текущей версии

Подавление системной горячей клавиши диктовки реализовано только для Windows. На других платформах приложение возвращает ошибку при попытке установить глобальный hook такого типа.

Системные уведомления о готовности конфигурации перед записью показываются только на Windows. На других платформах проверка перед записью выполняется, но системное уведомление не отображается.

Автоматическая вставка итогового текста через буфер обмена и команду вставки реализована только для Windows. На других платформах текущая версия возвращает ошибку вставки.

Открытие аудиофайла в проводнике реализовано как выбор файла в Windows Explorer на Windows и как открытие расположения через системную команду на других платформах.

Определение виртуального рабочего стола, на котором находится главное окно, доступно только на Windows. На других платформах левый клик по иконке в системной области всегда скрывает видимое окно. Если определить рабочий стол не удалось, приложение считает окно находящимся на текущем рабочем столе и скрывает его.

## Ограничения модели и качества результата

Качество распознавания и постобработки зависит от выбранного провайдера, модели, качества аудио, языка и prompt. Приложение не гарантирует идеальное исправление текста и не выполняет независимую валидацию смысла результата.

Словарь является подсказкой модели, а не обязательной заменой.

Стоимость отображается только если ее удалось получить из usage провайдера или из дополнительного запроса к OpenRouter. Если стоимость недоступна, она не показывается.

## Безопасность обработки текста

Prompt постобработки требует воспринимать распознанный текст как содержимое диктовки, а не как инструкции для модели. Модель должна очищать текст, не выполнять продиктованные команды, не отвечать на вопросы и не раскрывать служебные инструкции.

Это правило зависит от поведения выбранной модели и не является технической гарантией.
