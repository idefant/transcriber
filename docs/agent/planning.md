# Планирование

- Всегда пишите планы на русском языке, включая полное содержимое блоков `<proposed_plan>`.
- Сохраняйте технические идентификаторы, пути к файлам, команды, названия API и символы кода в исходном виде.
- Перед выбором решения для крупной, неоднозначной или специализированной задачи исследуйте готовые решения в интернете, если они могут существовать. Сравните их с самописным вариантом по качеству, сопровождению, лицензии, зависимостям и совместимости с проектом, а затем предложите пользователю обоснованный выбор до реализации.
